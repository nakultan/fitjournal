SECTION ONE

Executive summary

What's strong, what's leaking trust, and where the biggest design lift is.
★ STRENGTHS
  • Tokenised design system — colour, type, motion all swappable
  • Offline-first storage with multi-tier fallback + save-fail banner
  • Five rounds of post-audit fixes already shipped (P0–P3)
  • Per-set logging + Apple Health bridge w/o native code
  • WCAG 2.5.8 touch targets globalised (44px floor)
⚠ CAUTIONS
  • Today is still 1,228 lines — the most-used screen carries the most cognitive load
  • Progress mashes three modes (Overview / Exercises / History) into one segmented control with 6+ sections each
  • Streak & PR loops are reactive — no proactive reinforcement (no streak-save nudge, no PR teasing)
  • "Focused" Today layout is a preference hidden in Settings — most users will never find it
✗ AMATEUR TELLS
  • Pure black `#000` background on dark theme reads as "developer demo", not "premium app"
  • Every header has the same heft — no visual rhythm between hub-screens and tool-screens
  • Empty states use the same generic icon-title-button pattern across 5 screens (Jakob's Law inverted: too consistent)
  • Bottom nav is 5 evenly-weighted items — Today should dominate
  • No "first 30 seconds" celebration: log first set → silent toast. Should be a moment.

★ Reading the heatmap

The diagonal cluster of dark dots is on Recognition not recall · Hick · Cog-load · Behavioural reinforcement — all four spike on Today and Progress. These are the two screens carrying real aggregation work, and the design hasn't kept up.

Conversely Session and Settings are mostly clean — Session because it was rebuilt focused; Settings because P2.3 split it into cards. The lesson: shipped focused-mode work pays out.

Behavioural reinforcement is the worst column

Every row is dot-2 or dot-3. Translation: the app reliably records what happened but rarely makes the user feel something about it. The streak counter is a number on a card. PRs trigger confetti — once. Empty states say "no recipes yet" with a button. No anticipation, no near-miss feedback, no fresh-start moments.

Audit — by dimension

1 — Visual hierarchy

Hub card, weight banner, cardio, lifting, day-note all carry equal weight. Nothing on the screen says "your next action is here." Streak is the brightest stat but the user came to log a set — not admire a number.

2 — Information density · Hick

First viewport surfaces ~14 decisions: 3 nav arrows, 3 template chips, 5 stat cells, 4 add-affordances, the session button, the date picker. Choice paralysis at the moment of greatest mental commitment.

3 — Tap efficiency · Fitts

"Start session" — the most behaviourally valuable action — is a secondary chip beside the Lifting heading. Distance from thumb-rest (bottom-right) is maximum. It should be a primary CTA pinned bottom-right.

4 — Error prevention

P3.2 added soft warnings on weight/reps maxes. Good. Still missing: date picker lets you scroll to year 2099 then silently log. Hide future days entirely.

5 — Discoverability

Focused layout is a 50% information-density reduction that lives in Settings › Preferences. Nobody finds preference flags. Promote to a one-tap "Calm mode" toggle in the header.

6 — Motivation loops

Streak number sits there waiting to be looked at. No "save your streak" nudge at 22:00. No "you're one workout from your weekly goal" beat at 4/4 - 1. No fresh-start moment Monday morning. Reinforcement is reactive, not anticipatory.

7 — Empty states

P1.7 added the "Ready to train?" hero — already shipped. ✓

8 — Microinteractions

Reorder ▲▼ does the job but has no animation — list jumps. CountUp is used on summary; could also bump streak on first log of day for that "ratchet" feel.

9 — Accessibility

Date arrows have aria-labels, skip-link in place, 44px floor. ✓ Remaining gap: date label is a button but doesn't say aria-haspopup="dialog".

10 — Offline-first trust

SaveErrorBanner exists. But the absence of a banner doesn't mean "saved" — it means "nothing's wrong." P3.3 added "Auto-saved · 4s ago" in Settings → Data. That signal belongs on Today, not buried two screens away.

Audit — by dimension

✓ Strong: visual hierarchy

Rest timer, current exercise, next exercise — clearly ranked. Finish is the only green button. Good rhythm.

1 — Density

32/100 cognitive load — lowest in the app. Don't add anything.

3 — Tap efficiency · Fitts

Set chips are ~28pt wide; checkable area should be entire row, not just the checkmark. P3.x said .fj-session-check is 44×44 — ✓ but the surrounding row tap area still routes to "edit set" not "complete set". Decide on one default.

6 — Motivation

"Last time: 3×8 @ 180" is mentioned but small grey. This is the single most motivating piece of context in lifting apps. It should be the dominant subtitle of each exercise card, with a delta: "+5 lb vs last time".

8 — Microinteractions

Rest timer chimes ✓. But no visual rest countdown ring on the exercise card itself; user has to look at the pill. A faint progress-fill on the next set chip would tie countdown to context.

10 — Offline trust

Mid-session save fail would be catastrophic to user trust. Persisted check-state is fine, but a "saved 2s ago" micro-affordance on the Finish row would do real work.

★ Transformative idea — Train Mode

Session is so close to a "no thinking" mode. Hide the bottom nav while a session is active. Hide the back button behind a long-press. Treat it like a video call — the whole screen is the activity.

Audit — by dimension

1 — Visual hierarchy · Hick

Six top-level cards on Overview, each its own sub-system. The "Show charts & insights" disclosure (P2.5) was the right move — but it admits the screen has too much and hides it instead of restructuring it. Five+ stat blocks with no anchor.

6 — Recognition not recall

"Overview / Exercises / History" are three different mental models on one tab strip. After a week away, the user doesn't remember which tab held the PR list. Each needs a distinct visual identity AND a URL.

2 — Density

Tab strip + 4 stat tiles + Apple Health + weight chart + collapsed disclosure + heatmap. First scroll only. Treat Overview as a story, not a dashboard.

5 — Discoverability

"Exercises" tab is where strength progression lives — the most rewarding part of the screen. But the name doesn't promise that. "Records" or "Progress per exercise" would tell users what they're tapping for.

6 — Motivation loops

The Overview has every ingredient for a "progress story" — PR count, week-on-week tonnage, streak trajectory — but presents them as detached tiles. A single hero sentence ("You've moved 38,400 lb in the last 30 days — that's 12% more than last month.") would do more than five tiles.

7 — Empty states

Currently each tab has its own generic "no data yet" empty state. Good but undifferentiated.

8 — Microinteractions

Sparkline tooltips would close a real gap — you can see a trend but not read any specific point.

9 — Accessibility

P1.3 added aria-labels on Sparkline. ✓ The tab strip uses pill buttons; missing role="tablist" + role="tab" pattern. Keyboard users get nothing.

SECTION TEN

Top friction points

The ten things that, removed, change the feel of the app most.

1Today's hub does not start with the workout. The streak/goal card and weight banner sit above the lifting list. The thing the user came to do is below the thing they came to feel good about.

2"Start session" is a secondary chip. The single highest-value action lives next to a heading, sized like every other button. Should be a sticky FAB.

3Progress' tab strip hides three different mental models. Recognition fails on return. Needs three URLs + three distinct identities.

4Behavioural reinforcement is reactive. Reward fires after a PR. Nothing builds anticipation. Show "1 set away from a PR" before they lift.

5"Focused" layout is hidden in Settings. A 50%-density reduction nobody knows exists. Promote.

6Last-time numbers in Session are tiny grey. The single best motivator in lifting is "+5 lb vs last time" — and it's a 10pt muted caption.

7Bottom nav is five equally-weighted items. Today, Plan, and Settings get used daily / weekly / monthly. Same visual heft for all.

8Empty states all look the same. Icon + title + paragraph + button × 6 screens. Jakob's Law inverted — too consistent, no recognition cues.

9Recipes feels like a separate app. No bridge to the workout side. The reason this app has recipes at all is invisible.

10Auto-save signal is buried. P3.3 added it to Settings. Belongs on every screen — quiet, always there.

SECTION TWELVE

Prioritised backlog

P0 = ship before next release · P3 = ice box

P0 — ship this week

P0.1 Reverse Today's hierarchy. Lifts above hub; hub demoted to a single ambient line. Implement variant A behind a flag, expose via Tweaks.
P0.2 Promote "Start session" to a sticky FAB. Bottom-right, accent fill, 56pt — when today has exercises and is today.
P0.3 "Last time" deltas baked into Today's lift rows. +5 lb / +1 rep / same — coloured.
P0.4 Auto-save signal in PageHeader on every screen, not just Settings (move the 6px dot global).

P1 — next sprint

P1.1 Promote "Focused" → "Calm mode" — a one-tap header toggle, persisted; remove from Preferences as a buried flag.
P1.2 Progress: convert tab strip to three routes (#/progress/story · /records · /history); each gets a hero sentence on entry. Rename "Exercises" → "Records" while you're in there.
P1.3 "Next time, try X" recommendation on ExerciseDetail (Epley + progressive overload bump).
P1.4 PR-shot flag. When today's planned reps × weight exceeds the current PR by <2.5%, flag the row before the lift.
P1.5 Empty-state diversity. Each screen gets its own metaphor + copy. No more icon-title-button × 6.
P1.6 Distance-to-goal caption on ExerciseDetail goal pill — "20 lb to go · ~6 weeks at current rate". Turns a number into a path.
P1.7 Visually distinguish PR / e1RM / Goal on ExerciseDetail — they're past · projection · future, currently all the same pill style.
P1.8 First-set-of-day moment. First completed set ratchets the streak counter with CountUp + soft haptic. Tiny celebration, immediate reward.
P1.9 Hide future days in the date picker. Today's the max; the date arrow ▶ disables when on today. Error prevention.
P1.10 A11y polish bundle. role="tablist"/"tab" on Progress segmented; aria-haspopup="dialog" on Today's date label; tab-strip arrow-key nav.
P1.11 Session set row — pick one default. Whole-row tap = complete set (44pt floor already met); edit moves to long-press / explicit pencil. Stop routing the same tap to two intents.

P2 — design round

P2.1 Train Mode for Session. Hide bottom nav + back during active session; single-CTA layout.
P2.2 Recipes ↔ Today bridge. "Post-workout meal idea" card on Today; "Cook now" deep-link.
P2.3 Schedule rotated to "next 7 days from today" instead of Mon-Sun.
P2.4 Sparkline tooltips + per-point readout on tap.
P2.5 Streak-save nudge — push (via PWA notif if granted) at 19:00 if today's still empty and streak ≥ 5.
P2.6 Bottom nav adaptive — Session swaps to Pause / Finish; Summary state.
P2.7 Visual rest countdown as a thin progress ring around the active exercise card — ties the timer to its context, not just the floating pill.
P2.8 Templates collapse to a header row on Plan after first setup; weekly schedule becomes the primary surface (it's the repeat-use one).
P2.9 Seeded starter recipes on first install (3 high-protein basics) — mirrors the seeded Push/Pull/Legs templates and gets the empty Recipes screen out of zero-state.
P2.10 Protein bridge. Recipe macros + "log as eaten" → daily protein total visible on Today. Don't build a full nutrition page; build the link.
P2.11 Recipes filter pill — collapse the tag chip row behind a single "Filter" pill once the grid exceeds ~20 cards.
P2.12 Fresh-start Monday moment. First Today visit each Monday gets a one-line "New week — last week you trained 3×. Match it?" prompt that dismisses on first interaction.

P3 — ice box

P3.1 Drag-reorder for plan days.
P3.2 Recipe unit conversion (metric ↔ imperial).
P3.3 Hover-tooltips on desktop sparklines (mobile users get tap).
P3.4 Per-template colour / icon picker.
P3.5 "Backup reminder cadence" preference.
P3.6 Soften the dark theme — pure #000 bg reads as "developer demo." Bump to #0d0d10 or oklch-based near-black for a "premium app" feel.
P3.7 Page-header rhythm. Hub screens (Today, Progress) get a heavier display title; tool screens (Plan, Recipes, Settings) keep the current sub-display. Differentiate the role of each surface visually.
P3.8 Reorder ▲▼ animation. 150ms FLIP transition on swap — list currently jumps, breaking the connection between tap and result.
P3.9 Rotating Settings microcopy on the cards index — "1.2 MB on device" / "47 workouts saved" / "Last backup 12 days ago." Quiet trust reinforcement.
★ How to read priority · 32 items

P0 (4) moves the cognitive-load needle on the most-used screen. P1 (11) closes the behavioural-reinforcement & A11y gaps. P2 (12) is the design round — bold ideas that need user testing. P3 (9) is good-to-have polish that won't change feel but raises craft.

If you only ship four things from this audit, ship the four P0s. They turn Today from a hub into a workout screen. Every recommendation surfaced in the screen-by-screen audits above (sections 3–9) is represented here — nothing dropped on the floor.
